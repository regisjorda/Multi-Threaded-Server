In this assignment you are going to create a functional single threaded web server in the programming or scripting language of your choice (C or C++ recommended as you can then use the skeleton program linked to the bottom of this document). Your program will be capable of serving web pages to a browser such as Firefox, and compliant with HTTP/1.0 as defined by RFC 1945.

In this web server you must implement GET, HEAD, and POST methods of HTTP, and also your program must support text/plain and text/html content types. For POST, creation of a file on the server will suffice.

The following response codes must be implemented by your server program:
    - 200 (OK),
    - 201 (File created successfully - POST),
    - 400 (bad requests such as no leading / on path, HTTP/2.0+, no Content-Length for POST),
    - 403 (no read permissions),
    - 404 (file does not exist),
    - 501 (not implemented - ie not GET, HEAD, or POST methods that implemented in your program).

This web server will take a configuration file, myhttpd.conf, of the following format:
    HTTP1.0 [root]
    HTML html htm
where root is the document root, for example /var/www/html is the default directory for Apache on Linux. The other line defines file extensions for the file type of HTML which will be used in the server response.

Your server program requires following Command-line arguments.
    -p [port]
        specify port number to listen on
    -d
        print additional debugging information. Not specifically required, but this may be used to demonstrate PARTIAL functionality in the event that the server is not entirely functional.

    Testing: the telnet command is very useful for testing your web server. Try connecting to any web server, such as redhat.com by issuing this
command;
        telnet www.redhat.com 80
    where 80 is the port number (standard for web servers). When you have connected, try issuing a simple GET command,as the following;
        GET / HTTP/1.0 . For example:   GET /index.htm HTTP/1.0  requesting index.htm
    For HEAD, the format is almost identical (HEAD /indexi.htm HTTP/1.0) , except it uses HEAD keyword instead of GET. POST is slightly more complicated because the header must include the length of the data being sent. For that reason you should add Content-Length header as the following for POST :
       POST /filename HTTP/1.0
       Content-Length: decimal-size-in-bytes
    note: follow the header that is shown above, press Enter TWICE to inform the server that it has received the entire header. For a POST, after the header, press Enter twice, then input the same number of bytes (text) to be sent as specified in the header, then press Enter again to send.
